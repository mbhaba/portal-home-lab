# portal-home-lab

一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不服务于任何特定域名或网站，仅作为静态页面的集中存放与版本管理场所。

## 项目简介

`portal-home-lab` 旨在提供一个轻量、整洁的 HTML 页面集合。  
所有页面均为独立文件，互不依赖，可以单独浏览或部署。  
适合存放工具页面、信息展示页、实验性页面或个人收藏的静态内容。

## 目录结构

```
portal-home-lab/
├── docs/          # 页面归档目录
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/        # 共享资源（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md      # 本文件
└── LICENSE        # 许可证
```

## 页面归档说明

- 所有独立 HTML 页面统一存放于 `docs/` 目录下。
- 每个页面均为自包含的 HTML 文件，可单独打开或通过静态服务器托管。
- 页面之间无硬编码链接依赖，方便单独提取使用。
- 如有共享资源（如公共 CSS、JS 或图片），请放置在 `assets/` 目录中，并在页面中引用相对路径。

## 使用方式

1. 克隆本仓库：
   ```bash
   git clone https://github.com/your-username/portal-home-lab.git
   ```
2. 直接在浏览器中打开 `docs/` 下的任意 HTML 文件即可查看。
3. 也可将整个仓库部署到任意静态托管平台（如 GitHub Pages、Netlify 等），通过目录访问各个页面。

## 维护说明

- 新增页面：在 `docs/` 下创建新的 HTML 文件，并确保其独立可用。
- 修改页面：直接编辑对应文件，保持文件结构简洁。
- 删除页面：移除不再需要的文件，并清理 `assets/` 中无用的资源。
- 建议为每个页面添加简单的注释说明（如页面用途、创建日期等），便于日后管理。

## 贡献

欢迎提交 Pull Request 或 Issue 来改进页面内容或仓库结构。  
请确保提交的页面符合以下原则：
- 不包含恶意或侵权内容
- 独立可运行，无外部依赖（或依赖已放入 `assets/`）
- 不指向任何具体域名或商业网址
